    tree_classifier = tree.DecisionTreeClassifier(max_leaf_nodes = maxnode)

Avec sckit-learn, les performances en rgression sont values par dfaut via le score du $R^2$ (coefficient d'ajustement ou coefficient de dtermination). Ce score donne une mesure de la qualit de la prdiction :

$$ R^{2}=1- \frac {\sum _{i=1}^{n}(y_{i}-{\hat {y_{i}}})^{2}}{\sum _{i=1}^{n}(y_{i}-{\bar {y}})^{2}} = 1- \frac { \frac 1n \sum _{i=1}^{n}(y_{i}-{\hat {y_{i}}})^{2}}{ var (Y)}. $$

Un ajustement parfait correspond videmment $R^2 = 1$.

On reconnait un terme d'erreur quadratique au numrateur. On peut bien sr calculer le score sur l'chantillon d'apprentissage ou sur l'chantillon test.

Les performances de prvision sont values par dfaut dans sckit-learn via le score $R^2$. La dmarche standard est de calculer ce score par validation croise sur des chantillons tests obtenus par K-fold (score test). Pour les Bagged Tree et les Random Forest, il est possible de se passer de validation croise en calculant ce score par la mthode out-of-bag (score OOB).

Pour acclrer les calculs qui suivent (l'ide est de finir le TP en temps fini ...) on rduit la taille de la base de donnes d'apprentissage. Il est en fait courant de sous-chantillonner ainsi les bases de donnes dans la phase exploratoire d'une tude de Machine Learning, avant d'ajuster ensuite sur les bases de donnes compltes des prdicteurs adquats.
